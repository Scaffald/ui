# Button

Interactive button component with multiple variants and sizes. Supports icons, loading states, and accessibility features.

## Import

```tsx
import { Button } from '@scaffald/ui';
```

## Basic Usage

```tsx
import { Button } from '@scaffald/ui';

export default function Example() {
  return (
    <Button variant="filled" color="primary">
      Click Me
    </Button>
  );
}
```

## Props

See the [TypeScript definitions](https://github.com/Scaffald/ui/blob/main/src/components/Button/Button.tsx) for the complete props API.

### Common Props

All Button components accept standard React Native [`View` props](https://reactnative.dev/docs/view#props).

## Variants

### Visual Variants

- `filled` (default) - Solid background button
- `outline` - Outlined button with transparent background
- `ghost` - Minimal button with no background
- `text` - Text-only button

### Sizes

- `sm` - Small button
- `md` (default) - Medium button
- `lg` - Large button

### Loading state

Use the `loading` prop to show a loading indicator inside the button. The indicator is sized to fit the button (sm/md/lg). This matches the pattern used on onboarding and other forms.

**Do not** put a `<Spinner>` or other large indicator in the button's children—that can overflow the button. Use the `loading` prop instead.

If you don't see loading indicator updates after changing the UI package, rebuild it (`pnpm nx run @scaffald/ui:build`) or restart the dev server so the app picks up the latest build.

```tsx
<Button loading>Saving</Button>
<Button loading variant="filled" color="primary">Submit</Button>
```

<div className="flex-row-with-gap">
  <Button loading>Saving</Button>
  <Button loading variant="filled" color="primary">
    <span>Submit</span>
  </Button>
</div>
